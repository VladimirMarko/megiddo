# Use Production-Shaped Staging On Fly With Service-Owned SQLite Volumes

The first cloud deployment is a production-shaped staging environment: services run as separate deployable units with private service networking, durable persistence, and real token signing, but without the full operational burden of production uptime, backups, and horizontal scaling. We will keep Identity and Todo on service-owned SQLite files mounted on durable Fly.io volumes for this first deployment because it matches the current Persistence Adapter shape and gets deployment feedback before introducing a network database. The tradeoff is that stateful services start as single-writer instances; horizontal scaling and stronger database operations require a later move to a managed network database such as Postgres.
